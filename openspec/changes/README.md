# Changes Directory

This directory contains version history of specification changes.

## File Naming Convention

Format: `YYYY-MM-DD-brief-description.md`

Example: `2024-01-15-frontend-partials-structure.md`

## Change Log Template

```markdown
# Change Log: [Brief Description]

**Date**: YYYY-MM-DD
**Reason**: [Why this change was made]
**Affected Documents**: [List of affected spec documents]

## Changes
- [Description of change 1]
- [Description of change 2]

## Impact
[How this affects implementation]
```
